# Lessons

- 2026-03-20: セーブ移行を追加するときは、旧版からの移行だけでなく将来版 (`version > current`) を明示的に拒否し、初期化直後の自動保存で既存データを上書きしないことを検証する。
- 2026-03-21: 互換外セーブを保護するときは初回保存だけでなく、以後の `persist()` 全経路を read-only 化して、UI 操作一回で旧保存を失わないことを確認する。
- 2026-03-20: UI 操作を `gameOver` で一括停止する場合でも、デバッグやリセットのような管理操作は許可リストで別扱いにして、終幕後の検証導線を失わないようにする。
